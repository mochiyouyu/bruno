# Bruno 场景套件简版操作手册

## 这是什么

Runner 里现在有两种复用方式：

- `Scenario`：保存一组接口及其执行顺序。
- `Scenario Suite`：把多个场景串起来，作为一次自动化测试执行。

## 在哪里打开

1. 打开一个 Collection。
2. 点击顶部的 `Runner`。

## 怎么创建场景

1. 在右侧勾选要执行的接口。
2. 拖拽调整接口顺序。
3. 按需设置 `Delay` 和 `Tags`。
4. 点击 `Scenario` 区域的 `Save As`。
5. 输入名称并保存。

## 怎么把多个场景组合自动测试

1. 先保存好多个场景。
2. 在 `Scenario Suite` 区域里勾选要组合的场景。
3. 用上移、下移按钮调整场景顺序。
4. 点击 `Save As` 保存成套件。
5. 点击 `Run Suite` 执行整套自动测试。

## 自动化测试怎么用

最简单的使用方式是：

1. 先把一组接口分别整理成几个场景。
2. 再把这些场景组合成一个套件。
3. 以后直接运行这个套件，就能重复执行整套自动化测试。

一个常见示例：

- 场景 1：登录和鉴权检查
- 场景 2：核心业务接口检查
- 场景 3：结果校验和清理
- 套件：每日回归检查

以后要做回归时，只需要打开这个套件并点击 `Run Suite`。

## 套件执行时会怎样

- 系统会按套件里的场景顺序执行。
- 每个场景内部仍按自己保存时的接口顺序执行。
- 同一个接口如果出现在多个场景里，会执行多次。
- 结果列表里会显示每条请求属于哪个场景。

## 运行后怎么看结果

- 顶部会显示当前执行的是哪个套件。
- 每条请求下面会显示它属于哪个场景。
- 绿色表示通过，红色表示失败。
- 点击状态码可以看响应详情。
- 如果想再跑一遍，直接点击 `Run Again`。

## 怎么修改套件

1. 从 `Scenario Suite` 下拉框选择一个套件。
2. 勾选或取消勾选场景。
3. 调整顺序。
4. 点击 `Update` 保存修改。

如果想保留旧套件，再点 `Save As` 另存一个新套件。

## 怎么删除

- 删除场景：在 `Scenario` 区域选择后点 `Delete`。
- 删除套件：在 `Scenario Suite` 区域选择后点 `Delete`。

注意：删除场景后，引用它的套件会自动同步清理。

## 使用提示

- 只有已经保存到 Collection 的请求才能加入场景。
- 临时请求不会进入场景和套件。
- 改完场景或套件后，要点 `Update` 才会保存。
- 如果只是临时执行，也可以不保存套件，直接勾选场景后运行。
